# NameGuard

Плагин для блокировки ников и добавления дополнительных проверок для аккаунтов с правами.

- Поддерживает Paper и BungeeCord
- Удобные декларативные конфиги
- Проверка админов по IP, подсети или виртуальному хосту
- Блокировка никнеймов по полному совпадению или Regex строке
- Перезагрузка одной командой: `/nameguard reload`

## Блокировка

Все блокировки описываются декларативно в файле `blocked.yml`.

```yaml
# Название группы. Может быть каким угодно и будет отображаться в логах, если логирование включено
hackers:

  # Если выключено, группа будет игнорироваться
  enabled: true
  
  # Включить логирование при попытках захода на заблокированные ники
  log-to-console: false
  
  # Поддерживает MiniMessage
  kick-message: "<red>Hacking is not allowed!"
  
  # Заблокированные никнеймы. Не чувствительны к регистру
  nicknames:
    - meteor_client
    - liquid_bounce

  # Заблокированные паттерны. Не чувствительны к регистру
  # Используйте https://regex101.com для отладки
  patterns:
    - "meteor.*"
```

## Вайтлист

Все правила доступа описываются декларативно в файле `whitelist.yml`.

```yaml
# Название группы. Может быть каким угодно и будет отображаться в логах, если логирование включено
admins:

  # Если выключено, группа будет игнорироваться
  enabled: true

  # Включить логирование при неудачных попытках захода
  log-to-console: false

  # Поддерживает MiniMessage
  kick-message: "<red>Access denied."

  # Общие правила, применяющиеся для всех никнеймов в группе
  shared:
    - vhost:adminaccess.example.com
      
  nicknames:
    # Специальные условия для "owner". Домен "adminaccess.example.com" так же его пропустит
    owner:
      - subnet:127.0.0.1/24
      - ip:192.168.1.20
    # "admin" сможет зайти только используя домен "adminaccess.example.com"
    admin: []
```
